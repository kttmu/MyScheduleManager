## 4. リスク管理

### 4.1 リスク識別と対策

| リスク | 確度 | 影響 | 対策 |
|---|---|---|---|
| Excelフォーマット変更 | 中 | 中 | 仕様確定、サンプル作成 |
| メモリ不足（大規模WBS） | 中 | 高 | 早期にストレステスト実施 |
| グラフ生成ライブラリの互換性 | 低 | 中 | 複数候補検討、早期検証 |
| チーム人員不足 | 低 | 高 | スケジュール調整可能性確保 |
| 仕様追加要望 | 高 | 中 | 変更管理プロセス確立 |

### 4.2 対応計画

- **リスク監視**: 週1回のプロジェクト会議で監視
- **エスカレーション**: リスク発生時は即座にPMへ報告
- **バッファ**: スケジュールに10%のバッファを確保

## 9. 依存関係・制約事項

### 9.1 外部依存
- Python 3.8以上
- openpyxl / pandas
- Jinja2
- matplotlib（グラフ生成）

### 9.2 前提条件
- ExcelファイルはWBS形式である
- データは整形済みである
- ユーザーはPythonコマンド実行可能な環境を持つ

### 9.3 制約事項
- オンプレミス環境での使用
- ブラウザ: 最新版Chrome, Firefox, Safari対応
- 出力形式: HTML5
