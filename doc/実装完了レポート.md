## 🔑 実装の特徴

### 1. 期間加重平均による進捗率計算
```python
進捗率 = Σ(タスク進捗 × タスク期間) / Σ(タスク期間)
```
- 期間の長いタスクの進捗を優先的に反映
- より実測に近い進捗率の算出

### 2. ステータス値の統一
- **完了**: 進捗率 100%
- **着手中**: 進捗率 50%
- **未着手**: 進捗率 0%

### 3. スマートなステータス判定
- **On Track**: 実績 ≥ 計画
- **At Risk**: 計画-10% ≤ 実績 < 計画
- **Delayed**: 実績 < 計画-10% または 期限超過

### 4. 3層アーキテクチャ
```
入力 (Excel) 
  ↓
Data Layer (読み込み・構造化)
  ↓
Business Logic (計算・分析)
  ↓
Presentation Layer (可視化)
  ↓
出力 (HTML)
```

## 📈 計算例

### シナリオ例
```
テーマ: システム構築
┣ 成果物1: サーバ構築 (2026/1/15 - 2026/2/28, 45日)
┃ ┣ タスク1: OS インストール (2026/1/15 - 2026/1/31, 17日) - 完了 (100%)
┃ ┗ タスク2: ネットワーク設定 (2026/2/01 - 2026/2/28, 28日) - 完了 (100%)
┗ 成果物2: DB構築 (2026/3/01 - 2026/4/30, 61日)
  ┣ タスク3: スキーマ設計 (2026/3/01 - 2026/3/31, 31日) - 完了 (100%)
  ┗ タスク4: DB インスタンス (2026/4/01 - 2026/4/30, 30日) - 着手中 (50%)
```

### 成果物1の進捗計算
```
進捗 = (100% × 17日 + 100% × 28日) / (17日 + 28日)
     = (1700 + 2800) / 45
     = 4500 / 45
     = 100%
```

### 成果物2の進捗計算
```
進捗 = (100% × 31日 + 50% × 30日) / (31日 + 30日)
     = (3100 + 1500) / 61
     = 4600 / 61
     = 75.4%
```

### テーマ全体の進捗計算
```
進捗 = (100% × 45日 + 75.4% × 61日) / (45日 + 61日)
     = (4500 + 4599.4) / 106
     = 9099.4 / 106
     = 85.8%
```

## 📞 トラブルシューティング

### エラー: `ModuleNotFoundError: No module named 'openpyxl'`
**原因**: openpyxl がインストールされていない  
**解決策**:
```bash
pip install -r requirements.txt
```

### エラー: `ValueError: date_value は datetime オブジェクトまたは文字列である必要があります`
**原因**: Excelファイルの日付列が不正な形式  
**解決策**: 日付を以下の形式に統一
- YYYY-MM-DD（推奨）
- YYYY/MM/DD
- YYYY年MM月DD日

### エラー: `KeyError: '業務テーマ'`
**原因**: Excelファイルに必須列がない  
**解決策**: 以下の6列が必須:
1. 業務テーマ
2. 成果物
3. タスク
4. 開始時期
5. 終了時期
6. ステータス
