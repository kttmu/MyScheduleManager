# WorkScheduler プロジェクト - 基本設計概要

## 📋 プロジェクト概要

**WorkScheduler** は、Excel形式のWBS（Work Breakdown Structure）データを読み込み、進捗状況をJiraダッシュボード風のHTMLダッシュボードで可視化するPythonツールです。

---

## 🎯 主要機能

### 1. **データ入力**
- Excelファイルから業務計画データを読み込み
- WBS形式のデータ構造に対応（Theme→Deliverable→Task）

### 2. **進捗可視化**
- 計画進捗 vs 実績進捗をグラフで比較
- バーンダウンチャート表示
- テーマ別進捗率をテーブルで表示

### 3. **リスク管理**
- 遅延したタスク・成果物・テーマをリスト表示
- 進行中のタスク・成果物・テーマをリスト表示
- ステータス分類（On Track / At Risk / Delayed）

### 4. **ダッシュボードUI**
- Jiraダッシュボード相当のレスポンシブUI
- 複数のメトリクスウィジェット
- インタラクティブなテーブル表示

## 📊 ダッシュボード構成

### 5つのメインセクション

#### 1️⃣ **Key Metrics（キーメトリクス）**
- Overall Progress（進捗グラフ）: 計画 vs 実績
- Task Statistics（タスク統計）: 合計、完了、進行中、未着手
- Key Information（重要情報）: 更新時刻、遅延件数、ステータス

#### 2️⃣ **Burndown Chart（バーンダウンチャート）**
- 計画進捗と実績進捗の推移を時系列で表示
- トレンド分析

#### 3️⃣ **Delayed Items（遅延アイテム）**
- 遅延したテーマ・成果物・タスクをリスト表示
- 遅延日数を強調表示

#### 4️⃣ **In-Progress Items（進行中アイテム）**
- 進行中のテーマ・成果物・タスクをリスト表示
- 進捗率を表示

#### 5️⃣ **Theme Progress Overview（テーマ進捗概要）**
- テーマごとの進捗状況をテーブルで表示
- ステータス、進捗率、トレンドを表示

## ✨ 主な特徴

### 1. WBS対応
- Theme（テーマ）→ Deliverable（成果物）→ Task（タスク）の3階層構造
- Excel上では最初の行にのみテーマ・成果物名を記載

### 2. 進捗管理
- **計画進捗**: 開始日からの経過日数ベース
- **実績進捗**: 完了したタスク数ベース
- 両者の差分を「遅延」として検出

### 3. ステータス分類
- **On Track**: 実績進捗 ≥ 計画進捗
- **At Risk**: 計画進捗 - 10% ≤ 実績進捗 < 計画進捗
- **Delayed**: 実績進捗 < 計画進捗 - 10%、または期限超過

### 4. ユーザビリティ
- **レスポンシブデザイン**: モバイル対応
- **Jiraダッシュボード風UI**: 親しみやすいデザイン
- **複数グラフ対応**: 進捗バー、バーンダウン、円グラフ

## 📝 入力フォーマット

### Excelファイル構造

| 業務テーマ | 成果物 | タスク | 開始時期 | 終了時期 | ステータス |
|---|---|---|---|---|---|
| System Infrastructure | | | 2026-01-15 | 2026-04-30 | - |
| | Server Setup | | 2026-01-15 | 2026-02-28 | - |
| | | Install OS | 2026-01-15 | 2026-01-31 | 完了 |
| | | Configure Network | 2026-02-01 | 2026-02-28 | 着手中 |

**注意**: 
- WBS形式のため、テーマと成果物は各階層の最初の行にのみ記載
- Excelにはステータス列があり、タスク行に「完了」「着手中」「未着手」を記載
- 進捗率は計算で自動算出（手入力不要）

**ステータスと進捗率の対応**:
- 完了: 100%
- 着手中: 50%
- 未着手: 0%

**テーマ・成果物の進捗率計算**:
```
進捗率 = (各タスクの進捗率 × 各タスク期間) の合計 / 全タスク期間の合計
```
つまり、期間の長いタスクが全体進捗に与える影響が大きくなります。
