### 2.2 内部データモデル

#### Theme クラス
```python
class Theme:
    id: str                          # テーマID
    name: str                        # テーマ名
    start_date: datetime
    end_date: datetime
    deliverables: List[Deliverable]
    status: str                      # 'On Track', 'At Risk', 'Delayed'
    overall_progress: float          # 0-100%
    planned_progress: float          # 計画進捗
    actual_progress: float           # 実績進捗
```

#### Deliverable クラス
```python
class Deliverable:
    id: str
    name: str
    theme_id: str
    start_date: datetime
    end_date: datetime
    tasks: List[Task]
    
    @property
    def status(self) -> str:
        """含まれるタスクのステータスから判定"""
        if not self.tasks:
            return '未着手'
        completed = sum(1 for t in self.tasks if t.status == '完了')
        in_progress = sum(1 for t in self.tasks if t.status == '着手中')
        if completed == len(self.tasks):
            return '完了'
        elif in_progress > 0 or completed > 0:
            return '着手中'
        else:
            return '未着手'
    
    @property
    def overall_progress(self) -> float:
        """期間加重平均で進捗率を計算"""
        if not self.tasks:
            return 0.0
        total_duration = sum(t.duration_days for t in self.tasks)
        if total_duration == 0:
            return 0.0
        weighted_progress = sum(t.progress * t.duration_days for t in self.tasks)
        return (weighted_progress / total_duration)
    
    @property
    def planned_progress(self) -> float:
        """計画進捗（スケジュール進捗）"""
        total_days = (self.end_date - self.start_date).days
        elapsed_days = (datetime.now() - self.start_date).days
        if total_days <= 0:
            return 0.0
        return min(100.0, max(0.0, (elapsed_days / total_days) * 100))
```

#### Task クラス
```python
class Task:
    id: str
    name: str
    deliverable_id: str
    start_date: datetime
    end_date: datetime
    status: str                      # '完了', '着手中', '未着手'
    days_delayed: int                # 遅延日数
    
    @property
    def progress(self) -> float:     # 0-100%
        """ステータスに基づいて進捗率を返す"""
        if self.status == '完了':
            return 100.0
        elif self.status == '着手中':
            return 50.0
        else:  # '未着手'
            return 0.0
    
    @property
    def duration_days(self) -> int:  # タスク期間（日数）
        """タスク期間を日数で返す"""
        return (self.end_date - self.start_date).days + 1
```

#### Dashboard Data クラス
```python
class DashboardData:
```

## 5. 処理フロー設計

### 5.1 メイン処理フロー

```
START
  ↓
1. Excel ファイル読み込み
  ├─ ファイル存在確認
  ├─ Excelパース
  └─ 行データ抽出
  ↓
2. データバリデーション
  ├─ 必須フィールド確認
  ├─ 日付形式チェック
  └─ WBS構造妥当性チェック
  ↓
3. WBSパース処理
  ├─ Theme抽出・生成
  ├─ Deliverable抽出・生成
  ├─ Task抽出・生成
  └─ 階層関係構築
  ↓
4. 進捗計算処理
  ├─ 計画進捗 = (Today - Start) / (End - Start)
  ├─ 実績進捗 = Completed Tasks / Total Tasks
  └─ テーマ別集計
  ↓
5. データ分析処理
  ├─ 遅延検出 (Today > End Date and Progress < 100%)
  ├─ ステータス分類
  ├─ リスク評価
  └─ 統計情報生成
  ↓
6. HTMLレンダリング
  ├─ グラフ生成 (SVG)
  ├─ テーブルレンダリング
  ├─ CSS/JavaScript統合
  └─ レスポンシブ対応
  ↓
7. HTMLファイル出力
  ↓
END
```

### 5.2 進捗率計算ロジック

#### タスク進捗
```
Task Progress (%):
  - ステータス = '完了' → 100%
  - ステータス = '着手中' → 50%
  - ステータス = '未着手' → 0%
```

#### 計画進捗（スケジュール進捗）
```
Planned Progress (%) = 
    max(0, min(100, (Today - Start Date) / (End Date - Start Date) * 100))
```

#### テーマ・成果物の進捗（期間加重平均）
```
Item Progress (%) = 
    (Sum of (Task Progress × Task Duration)) / (Sum of Task Duration) × 100

ここで:
  Task Duration = End Date - Start Date (日数)
  Sum of Task Duration = 全タスクの期間の総和（基準値=100)
```

### 5.3 ステータス分類ロジック

```
Status Classification:
  IF Overall Progress >= Planned Progress:
    Status = "On Track" (Green)
  ELSE IF Overall Progress >= Planned Progress - 10%:
    Status = "At Risk" (Yellow)
  ELSE:
    Status = "Delayed" (Red)

また、以下の条件も加える：
  IF Today > End Date AND Overall Progress < 100%:
    Status = "Delayed" (Red)

ここで:
  Overall Progress = 期間加重平均進捗率
  Planned Progress = スケジュール進捗率
```

### 5.4 遅延検出ロジック

```
FOR each Task:
  IF Today > End Date AND Status != '完了':
    Delay Status = True
    Days Delayed = Today - End Date
    Priority = High

FOR each Deliverable/Theme:
  IF Today > End Date AND Overall Progress < 100%:
    Delay Status = True
    Days Delayed = Today - End Date
    Priority = High
```
